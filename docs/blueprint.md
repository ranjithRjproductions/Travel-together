## Core Features:

- Signup: User enters name, email, password, selects role (Traveler / Guide). Validate all fields. Create account (mock / placeholder). Store role with account. Redirect to Login page.
- Login: User enters email + password. Authenticate (mock). Fetch stored role. If role === Traveler -> /traveler/dashboard. If role === Guide -> /guide/dashboard.
- Traveler Dashboard: Display the Traveler dashboard based on authentication and role.
- Guide Dashboard: Display the Guide dashboard based on authentication and role.

## Style Guidelines:

- Primary color: Vibrant blue (#4285F4) to evoke a sense of trust and adventure.
- Background color: Light gray (#F0F4F8) for a clean, modern look.
- Accent color: Yellow-orange (#FFB300) for call-to-action buttons and highlights, representing warmth and energy.
- Body and headline font: 'PT Sans', sans-serif, for readability and a modern, humanist feel.
- Use location-based and travel-themed icons to represent locations and the traveler/guide role.
- Design a clear, intuitive layout to promote the distinction between a traveler and a guide, using cards with profile and activity summaries.
- Add subtle transition animations between pages and interactive elements, to guide users between the key states in each of the traveler and guide roles.
